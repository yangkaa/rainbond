> 版本: v3.0
> 更新时间: 2026-01-13
> 状态: 讨论确认

## 一、战略目标

### 1.1 核心目标

**让开源用户能自助安装企业插件，实现试用→付费的商业转化**

当前痛点：
- 开源用户无法自助安装企业插件（需要人工配置）
- 授权流程繁琐（导出集群信息 → 人工生成授权码 → 填入平台）
- 前端资源无版本管理，无法升级
- 无法获取用户信息，无法运营
- 插件下载和复制无法控制

目标状态：
- 开源用户 10 分钟内完成安装和试用激活
- 注册即可试用，在线自动授权
- 试用到期自动提醒，顺畅转化为付费用户
- 企业客户可离线部署和维护
- 插件下载需授权，复制到其他环境无法使用

### 1.2 核心原则

1. **开源优先**：先服务开源用户，通过体验驱动转化
2. **自助为主**：减少人工介入，用户能自己完成全流程
3. **安全可控**：授权机制防止未授权使用和复制
4. **渐进增强**：先实现核心流程，再优化体验

### 2.2 授权模型

**双层时间控制：**

| 层级 | 字段 | 作用 | 过期后果 |
|------|------|------|----------|
| 授权期限 | expire_at | 控制功能可用性 | 平台功能受限 |
| 订阅期限 | subscribe_until | 控制下载/更新 | 无法安装新插件或升级 |

**版本控制：**

| 版本 | 可安装插件 |
|------|-----------|
| 基础版 | enterprise-base |
| 高级版 | 全部企业插件 |

### 2.3 授权信息结构

```go
type LicenseToken struct {
    // 基础信息
    ClusterID      string   `json:"cluster_id"`       // 集群唯一标识 (kube-system UID)
    Tier           string   `json:"tier"`             // basic / advanced

    // 时间控制
    ExpireAt       int64    `json:"expire_at"`        // 授权到期时间
    SubscribeUntil int64    `json:"subscribe_until"`  // 订阅到期时间

    // 权限控制
    AllowedPlugins []string `json:"allowed_plugins"`  // 允许安装的插件列表

    // 资源限制
    NodeLimit      int      `json:"node_limit"`       // 节点数限制
    MemoryLimit    int64    `json:"memory_limit"`     // 内存限制

    // 签名 (RSA-SHA256)
    Signature      string   `json:"signature"`        // 私钥签名，公钥验证
}
```

### 2.4 防复制机制

1. **集群绑定**：授权码包含 cluster_id，只在指定集群有效
2. **签名验证**：用私钥签名，插件内置公钥验证，无法伪造
3. **AccessKey 绑定**：下载插件需要 AccessKey，绑定镜像仓库凭证

### 2.5 授权流程

```
阶段1: 安装授权插件 (免费)
────────────────────────
用户安装「企业授权插件」→ 显示"未授权，请联系获取授权"
                       → 显示集群 ID 供用户提供给销售

阶段2: 获取授权
──────────────
用户联系销售 → 提供集群 ID → 销售生成授权码 (私钥签名)
           → 用户填入授权码 → 授权插件验证签名
           → 验证成功 → 写入 ConfigMap + 获得 AccessKey

阶段3: 安装企业插件
─────────────────
用户点击安装 → 使用 AccessKey 下载插件包
           → 插件启动时读取 ConfigMap
           → 用内置公钥验证签名
           → 检查 cluster_id、plugin_id、expire_at
           → 验证通过 → 正常运行

阶段4: 过期处理
─────────────
订阅过期 → AccessKey 失效 → 无法下载/更新，已安装可用
授权过期 → 平台功能受限 → 需续费
```

## 四、M1: 自助安装

### 4.1 目标

**开源用户能在 10 分钟内完成企业插件的安装和授权激活**

### 4.2 验收标准

- [ ] 企业授权插件可免费安装
- [ ] 安装后显示集群 ID，引导用户获取授权
- [ ] 用户可输入授权码并成功激活
- [ ] 激活后可安装企业插件
- [ ] 企业插件启动时验证授权（签名、集群、权限）
- [ ] 未授权时企业插件显示锁定
- [ ] 授权过期时平台功能受限

### 4.3 核心功能模块

| 模块 | 功能点 |
|------|--------|
| 企业授权插件 | 授权码验证、授权信息存储、AccessKey 管理 |
| 插件中心页面 | 插件列表、状态展示、安装/升级/卸载 |
| 授权管理 | 授权状态栏、授权码输入、激活反馈 |
| 插件安装机制 | AccessKey 下载、镜像拉取、进度展示 |
| 插件自验证 | 内置公钥、签名校验、权限检查 |
| 过期处理 | 订阅过期、授权过期、续费提醒 |

### 4.4 技术方案要点

**授权验证：非对称加密**
- 你们持有私钥，生成签名
- 插件内置公钥，验证签名
- 无法伪造授权

**前端保护：嵌入后端**
- 前端 JS 打包到 Go 二进制
- 请求时先验证授权再返回
- 防止替换前端代码

**服务发现：ConfigMap**
- 授权信息写入固定 ConfigMap
- 其他插件直接读取验证
- 不依赖授权插件是否在线

- [ ] 企业授权插件

- [ ] 企业插件改造（内置公钥、自验证）
